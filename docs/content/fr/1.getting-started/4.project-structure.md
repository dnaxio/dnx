# Quickstart

## 1. Initialiser un projet

```bash
mkdir my-app && cd my-app
dnx init --name my-app
```

Cela crée un fichier `.dnax/dnx.yaml` avec une configuration de base.

## 2. Configurer vos serveurs

Éditez `.dnax/dnx.yaml` pour renseigner vos serveurs :

```yaml
environments:
  staging:
    servers:
      - name: staging-1
        host: 10.0.1.10
        user: deploy
        port: 22
        key_path: ~/.ssh/id_ed25519
```

## 3. Définir votre workload

```yaml
workloads:
  - name: web
    driver: flox
    start_cmd: "bun run server.ts"
    ports:
      - 3000
    env:
      NODE_ENV: "production"
      DATABASE_URL: "${secret:DATABASE_URL}"
    build:
      steps:
        - run: bun install
        - run: bun run build
    health:
      type: http
      endpoint: /health
```

## 4. Configurer les secrets

```bash
dnx secrets init
dnx secrets set DATABASE_URL=postgres://user:pass@host:5432/db --env production
```

## 5. Déployer

```bash
# Build (exécute les étapes de build via flox)
dnx build web

# Déployer en staging
dnx deploy web --env staging

# Déployer en production (blue-green)
dnx deploy web --env production --strategy blue-green
```

## 6. Vérifier

```bash
# Statut du workload
dnx workload status web --env staging

# Logs
dnx workload logs web --env staging --follow

# Health check
dnx health check web --env staging
```
