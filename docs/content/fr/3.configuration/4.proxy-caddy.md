# Proxy & Caddy

DNX intègre Caddy comme reverse proxy et load balancer. La configuration est générée automatiquement depuis votre `dnx.yaml`.

## Configuration

```yaml
proxy:
  auto_ssl: true
  email: admin@example.com    # Pour Let's Encrypt
  routes:
    - domain: app.example.com
      target: web
      port: 3000
      lb_policy: round_robin
      ssl: true
      headers:
        X-Frame-Options: DENY
```

## Commandes

```bash
# Démarrer Caddy
dnx proxy start --env staging

# Redémarrer (graceful, zero-downtime)
dnx proxy restart

# Arrêter
dnx proxy stop

# Statut
dnx proxy status --env production

# Logs
dnx proxy logs server-name --follow
```

## Load Balancing

```yaml
routes:
  - domain: api.example.com
    target: api
    port: 8080
    lb_policy: least_conn    # round_robin | least_conn | first | ip_hash
    ssl: true
```

Les politiques disponibles :
- `round_robin` — Distribution équitable
- `least_conn` — Moins de connexions actives
- `first` — Premier serveur disponible
- `ip_hash` — Sticky sessions par IP

## Gestion des routes

```bash
# Lister les routes
dnx proxy route list

# Ajouter une route
dnx proxy route add app2.example.com web --port 3001

# Supprimer
dnx proxy route remove app2.example.com
```

## SSL

Caddy gère automatiquement les certificats SSL via Let's Encrypt. Aucune configuration supplémentaire n'est nécessaire si le domaine pointe vers le serveur.

```bash
# Statut des certificats
dnx proxy ssl
```

## Headers personnalisés

```yaml
routes:
  - domain: app.example.com
    headers:
      X-Frame-Options: DENY
      X-Content-Type-Options: nosniff
      Content-Security-Policy: "default-src 'self'"
```

## Rate Limiting

```yaml
proxy:
  global_options:
    rate_limit:
      rate: 100
      burst: 50
      period: 1s
```
