## Composants

### CLI (Bun + TypeScript)

Le CLI s'exécute localement sur la machine du développeur. Toute la logique est embarquée dans un seul processus Bun.

### SSH Engine

Basé sur `ssh2` (Node.js). Pool de connexions avec :
- Limite de concurrence configurable (défaut: 10)
- Retry automatique (exponential backoff)
- Keep-alive pour les connexions longues
- SFTP pour les transferts de fichiers

### SQLite (state.db)

Stocké dans `~/.dnx/state.db`. Contient :
- Déploiements
- Releases (pour rollback)
- Locks
- Health check logs
- Secrets chiffrés
- Configuration des serveurs

### Agent (optionnel)

L'agent `dnxd` est un binaire TypeScript compilé via `bun build --compile`. Il s'exécute sur les serveurs et expose une API JSON-RPC sur `localhost:9876`.

Méthodes disponibles : `ping`, `status`, `deploy`, `start`, `stop`, `restart`, `logs`, `processInfo`.

## Modes de déploiement

### Agentless (défaut)

Toutes les opérations passent par SSH directement. Le CLI envoie des scripts shell et les exécute à distance.

### Agent

Le CLI communique avec l'agent via HTTP (tunnel SSH ou direct). L'agent gère la supervision des processus, le buffer de logs et les restart policies.
