# Agent dnxd

L'agent `dnxd` est un processus léger qui s'exécute sur les serveurs pour améliorer les performances (buffer de logs, supervision de processus, pas de latence SSH par commande).

## Installation

```bash
dnx agent install staging-1
```

L'agent est un binaire TypeScript compilé avec `bun build --compile`. Il est uploadé via SSH et démarré sur le serveur.

## Statut

```bash
# Vérifier un serveur
dnx agent status staging-1
# staging-1: RUNNING

# Tous les serveurs
dnx agent status
```

## Désinstallation

```bash
dnx agent uninstall staging-1
```

## API (JSON-RPC)

L'agent écoute sur `localhost:9876` et expose :

| Méthode | Description |
|---------|-------------|
| `ping` | Test de connectivité |
| `status` | Statut de l'agent + processus managés |
| `deploy` | Déploie une nouvelle version |
| `start` / `stop` / `restart` | Cycle de vie |
| `logs` | Buffer circulaire de logs (500 dernières lignes) |
| `processInfo` / `processes` | Infos sur les processus |

## Mode agentless vs agent

| | Agentless | Agent |
|---|---|---|
| **Latence** | 1 requête SSH par commande | HTTP local, quasi-instantané |
| **Logs** | `tail -f` via SSH | Buffer circulaire en mémoire |
| **Restart policy** | Non (process meurt = fin) | Oui (always, on-failure) |
| **Installation** | Aucune | `dnx agent install` |

**Recommandé** : Agentless pour le développement/staging, agent pour la production.
