# dnx build

```bash
dnx build <workload> [options]
```

Exécute les étapes de build **locales** (`build.local.steps`) sur la machine hôte. Aucun driver (flox, docker, etc.) n'est impliqué — les commandes tournent directement dans le dossier du workload.

## Options

| Option | Description | Défaut |
|--------|-------------|--------|
| `--env <env>` | Environnement cible | `staging` |
| `--no-cache` | Ignore le cache de build | `false` |
| `--skip-tests` | Sauter les tests | `false` |

## Exemple

```bash
dnx build web
```

Exécute les étapes de build locales définies dans `dnx.yaml` :

```yaml
workloads:
  - name: web
    build:
      local:
        steps:
          - run: bun generate
          - run: bun build
```

Les étapes tournent directement sur la machine hôte, dans le dossier spécifié par `sync.source` (ou `.` par défaut). Les outils (`bun`, `npm`, `node`, etc.) doivent être installés localement.

## Build local dans le déploiement

`dnx deploy` exécute automatiquement les étapes de build local **avant** de synchroniser les sources et de déployer. Si `build.local.steps` est vide, l'étape est simplement ignorée.

```
Ordre : build.local → sync → build.server → start container
```
