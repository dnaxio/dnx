# dnx build

```bash
dnx build <workload> [options]
```

## Options

| Option | Description | Défaut |
|--------|-------------|--------|
| `--env <env>` | Environnement cible | `staging` |
| `--no-cache` | Ignore le cache de build | `false` |
| `--skip-tests` | Sauter les tests | `false` |

## Build flox

```bash
dnx build web
```

Exécute les étapes de build définies dans `dnx.yaml` via `flox activate` :

```yaml
workloads:
  - name: web
    build:
      steps:
        - run: npm ci
        - run: npm run build
        - run: npm test     # Sauté avec --skip-tests
```

## Build OCI

Si le runtime est `docker` ou `podman`, `dnx build` utilise BuildKit pour construire l'image :

```bash
dnx build web --no-cache
```

Équivalent à :

```bash
docker buildx build -f Dockerfile -t ghcr.io/org/web:latest .
```
