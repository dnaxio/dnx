# Health Checks

```bash
dnx health check [workload] [options]
```

## Types de sondes

### HTTP

```yaml
health:
  type: http
  endpoint: /health
  timeout: 5s
  retries: 3
```

### TCP

```yaml
health:
  type: tcp
  endpoint: localhost:3000
  timeout: 3s
```

### Command

```yaml
health:
  type: command
  command: "curl -sf http://localhost:3000/health"
```

## Commandes

```bash
# Vérification ponctuelle
dnx health check web --env production

# Surveillance continue (toutes les 30s)
dnx health watch web --env production --interval 30

# Historique
dnx health history --workload web
```

## États

| État | Déclencheur |
|------|------------|
| 🟢 `healthy` | Tous les checks OK |
| 🟡 `degraded` | 1-2 échecs consécutifs |
| 🔴 `unhealthy` | 3+ échecs consécutifs |

Les transitions sont logguées dans SQLite et affichées dans le terminal.
