# Rollback

```bash
dnx rollback <workload> [--tag <tag>] [--env <env>]
```

## Rollback standard

Revient au déploiement précédent :

```bash
dnx rollback web --env production
```

## Rollback vers un tag spécifique

```bash
dnx rollback web --tag v1.2.3
```

## Lister les tags disponibles

```bash
dnx rollback web
# Tags for web on prod-1:
#   web:staging-1718143200000 ...
#   web:staging-1718056800000 ...
```

## Fonctionnement

1. Trouve le tag cible dans les images Docker
2. Stop le workload en cours
3. Re-tag l'image cible comme `latest`
4. Redémarre le workload
5. Met à jour Caddy si nécessaire

Le rollback est basé sur les tags Docker, les releases précédentes doivent être disponibles sur le(s) serveur(s).
