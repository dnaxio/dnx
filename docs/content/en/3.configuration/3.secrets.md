# Secrets

DNX encrypts your secrets with AES-256-GCM and auto-injects them at deploy time.

## Initialize

```bash
dnx secrets init
```

Generates a master key at `~/.dnx/master.key` (chmod 600).

## Set a secret

```bash
dnx secrets set DATABASE_URL=postgres://user:pass@host:5432/db --env production
```

## List secrets

```bash
dnx secrets print
dnx secrets print --json
```

## Extract secrets

```bash
dnx secrets extract --env production
dnx secrets extract --env production --format json
```

## Usage in dnx.yaml

```yaml
runtime:
  env:
    DATABASE_URL: "${secret:DATABASE_URL}"
```

## Security

- Master key: `~/.dnx/master.key` (chmod 600)
- Alternative: `DNX_MASTER_KEY` environment variable
- **Never commit `master.key` to Git**
- Secrets are encrypted at rest in SQLite
