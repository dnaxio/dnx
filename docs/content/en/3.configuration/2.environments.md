# Environments

DNX supports three environments: `staging`, `test`, and `production`.

## Definition

```yaml
environments:
  staging:
    servers:
      - name: staging-1
        host: 10.0.1.10
    deploy_strategy: rolling

  production:
    servers:
      - name: prod-1
        host: 10.0.10.1
      - name: prod-2
        host: 10.0.10.2
    deploy_strategy: blue-green
```

## Per-environment overrides

```
dnx.yaml           # Base config
dnx.staging.yaml   # Staging overrides
dnx.production.yaml # Production overrides
```

## Deploy Strategies

| Strategy | Description | Use case |
|----------|------------|----------|
| `rolling` | Deploy server by server | Staging, small deploys |
| `blue-green` | New set → switch → old destroyed | Production, zero-downtime |
| `canary` | Subset deploy, verify, propagate | Production, risky deploys |

## Diff between environments

```bash
dnx config diff staging production
```
