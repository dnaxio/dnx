# Architecture

```
CLI (Bun + TypeScript)
├── Config (YAML + Zod)
├── Secrets (AES-256-GCM)
├── SSH Pool (ssh2, parallel, retry)
├── SQLite (state.db)
└── Core Engine
    ├── Deploy (rolling / blue-green / canary)
    ├── Build (flox + OCI)
    ├── Rollback
    └── Health Checks
```

## Components

- **CLI**: Bun + TypeScript, runs locally, single binary
- **SSH Engine**: `ssh2` with concurrency limit (default 10), retry, keep-alive, SFTP
- **SQLite**: `~/.dnx/state.db` — deployments, releases, locks, health logs, secrets
- **Agent (optional)**: TypeScript compiled via `bun build --compile`, JSON-RPC on `localhost:9876`

## Modes

- **Agentless** (default): All operations via SSH. CLI sends shell scripts.
- **Agent**: CLI communicates via HTTP tunnel through SSH. Agent handles process supervision and log buffering.
