# Lock System

DNX uses a lock system to prevent two concurrent deployments on the same resource.

## Commands

```bash
dnx lock list
dnx lock release deploy:web:production
```

## Lock Types

| Resource | Description |
|----------|-------------|
| `deploy:global` | One deploy at a time (global) |
| `deploy:<workload>:<env>` | One deploy per workload/env |
| `server:<name>` | Server-level lock |
| `proxy:reload` | Exclusive Caddy reload |

## How it works

- **Local lock**: SQLite — prevents two concurrent CLIs on the same machine
- **Remote lock**: `/tmp/dnx-lock-<resource>.lock` on server
- **TTL**: Locks auto-expire after 30 min (configurable)
- **Cleanup**: `Bun.cron()` periodically cleans expired locks
