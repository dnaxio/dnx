# Agent dnxd

The `dnxd` agent is a lightweight process running on servers for improved performance.

## Install

```bash
dnx agent install staging-1
```

## Status

```bash
dnx agent status staging-1
```

## Uninstall

```bash
dnx agent uninstall staging-1
```

## API (JSON-RPC)

Listens on `localhost:9876`:

| Method | Description |
|--------|-------------|
| `ping` | Connectivity test |
| `status` | Agent + managed processes |
| `deploy` | Deploy new version |
| `start` / `stop` / `restart` | Lifecycle |
| `logs` | Circular log buffer (500 last lines) |

## Agentless vs Agent

| | Agentless | Agent |
|---|---|---|
| **Latency** | 1 SSH request per command | Local HTTP, instant |
| **Logs** | `tail -f` via SSH | In-memory circular buffer |
| **Restart policy** | No | Yes (always, on-failure) |
| **Installation** | None | `dnx agent install` |

**Recommended**: Agentless for dev/staging, agent for production.
