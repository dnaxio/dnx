# Docker & OCI

DNX supports OCI containers (Docker/Podman) alongside flox.dev.

## Configuration

```yaml
workloads:
  - name: web
    runtime:
      type: docker
      dockerfile: ./Dockerfile
      registry: ghcr.io/org/web
      ports: [3000]
```

## Commands

```bash
dnx oci build web --tag v1.2.3
dnx oci push web
dnx oci pull web prod-1
dnx oci images prod-1
dnx oci cleanup prod-1 --keep 5
```

## BuildKit without Docker

DNX uses **BuildKit** (`buildctl`) to build OCI images **without a Docker daemon**:

```bash
brew install buildkit   # macOS
apt install buildkit    # Linux
dnx oci build web
```

## Podman (rootless)

```yaml
runtime:
  type: podman
```

## Auto Dockerfile Generation

If no Dockerfile exists, DNX auto-generates one based on project type:
- Node.js, Python, Go, Rust — multi-stage builds.
