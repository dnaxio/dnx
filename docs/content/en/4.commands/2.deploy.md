# dnx deploy

```bash
dnx deploy <workload> [options]
```

## Options

| Option | Description | Default |
|--------|-------------|---------|
| `--env <env>` | Target environment | `staging` |
| `--strategy <s>` | rolling, blue-green, canary | From dnx.yaml |
| `--dry-run` | Simulate without executing | `false` |
| `--no-health-check` | Skip health check | `false` |
| `--timeout <s>` | SSH timeout (seconds) | `30` |

## Examples

```bash
# Simple staging deploy
dnx deploy web --env staging

# Production with blue-green strategy
dnx deploy web --env production --strategy blue-green

# Dry-run (validate without deploying)
dnx deploy web --env production --dry-run

# Deploy all workloads
dnx deploy --all --env production --tag v1.2.3
```

## Deployment Cycle

1. **Local build** — runs `build.local.steps` on the host machine (if configured)
2. **Prerequisites check** — flox, caddy installed on servers
3. **Lock acquisition** — prevents concurrent deploys
4. **Sync** — syncs source files to servers
5. **Server build** — runs `build.server.steps` on remote servers (if configured)
6. **Release** — creates new release (`$HOME/.dnx/workloads/<name>/releases/<version>`)
7. **Start** — starts new version
8. **Health check** — verifies workload responds
9. **Proxy** — updates Caddy
10. **Lock release**
