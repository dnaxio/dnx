# dnx deploy

```bash
dnx deploy <workload> [options]
```

## Options

| Option | Description | Default |
|--------|-------------|---------|
| `--env <env>` | Target environment | `staging` |
| `--strategy <s>` | rolling, blue-green, canary | From dnx.yaml |
| `--dry-run` | Simulate without executing | `false` |
| `--no-health-check` | Skip health check | `false` |
| `--timeout <s>` | SSH timeout (seconds) | `30` |

## Examples

```bash
dnx deploy web --env staging
dnx deploy web --env production --strategy blue-green
dnx deploy web --env production --dry-run
```

## Deployment Cycle

1. **Prerequisites check** — flox, caddy installed on servers
2. **Lock acquisition** — prevents concurrent deploys
3. **Build** — runs configured build steps
4. **Stop** — stops running workload
5. **Release** — creates new release
6. **Start** — starts new version
7. **Health check** — verifies workload responds
8. **Proxy** — updates Caddy
9. **Lock release**
