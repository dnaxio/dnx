# dnx build

```bash
dnx build <workload> [options]
```

Runs the **local** build steps (`build.local.steps`) on the host machine. No driver (flox, docker, etc.) is involved — commands run directly in the workload directory.

## Options

| Option | Description | Default |
|--------|-------------|---------|
| `--env <env>` | Target environment | `staging` |
| `--no-cache` | Ignore build cache | `false` |
| `--skip-tests` | Skip tests | `false` |

## Example

```bash
dnx build web
```

Runs the local build steps defined in `dnx.yaml`:

```yaml
workloads:
  - name: web
    build:
      local:
        steps:
          - run: bun generate
          - run: bun build
```

Steps run directly on the host machine, in the directory specified by `sync.source` (or `.` by default). Tools (`bun`, `npm`, `node`, etc.) must be installed locally.

## Local build in deployment

`dnx deploy` automatically runs local build steps **before** syncing sources and deploying. If `build.local.steps` is empty, the step is simply skipped.

```
Order: build.local → sync → build.server → start container
```
