# Introduction

DNX Deploy is a multi-server deployment tool built on SSH, designed for teams who want a simple alternative to Docker Swarm, Kubernetes, or Capistrano.

## Why DNX?

| Need | DNX | Docker Swarm | Kubernetes | Capistrano |
|------|-----|-------------|------------|------------|
| Simple (single binary) | ✅ | ❌ | ❌ | ❌ (Ruby) |
| Multi-server SSH | ✅ 50 servers | ❌ | ✅ | ✅ |
| Agentless | ✅ | ❌ | ❌ | ✅ |
| Reproducible envs | ✅ flox + OCI | ✅ Docker | ✅ OCI | ❌ |
| Built-in load balancer | ✅ Caddy | ✅ | ✅ | ❌ |
| Preview envs per PR | ✅ | ❌ | ❌ (3rd party) | ❌ |
| Learning curve | Low | Medium | High | Medium |

## Tech Stack

```
┌─────────────────────────────────────────┐
│ CLI (Bun + TypeScript)                   │
│ ┌───────┐ ┌──────┐ ┌──────┐ ┌────────┐ │
│ │Config │ │Secrets│ │Build │ │Deploy  │ │
│ │YAML   │ │AES   │ │flox  │ │Engine  │ │
│ └───────┘ └──────┘ │OCI   │ └────────┘ │
│                    └──────┘             │
│ ┌──────────────────────────────────────┐│
│ │ SSH Pool (ssh2, parallel, retry)     ││
│ └──────────────────────────────────────┘│
│ ┌──────────────────────────────────────┐│
│ │ SQLite (state.db, local)             ││
│ └──────────────────────────────────────┘│
└────────────────┬────────────────────────┘
                 │ SSH
     ┌───────────┼───────────┐
     ▼           ▼           ▼
┌─────────┐ ┌─────────┐ ┌─────────┐
│Server A │ │Server B │ │Server C │
│ flox    │ │ Docker  │ │ Caddy   │
│ Caddy   │ │ Caddy   │ │ flox    │
│ Workload 1   │ │ Workload 2   │ │ Workload 1   │
└─────────┘ └─────────┘ └─────────┘
```

- **CLI**: Bun + TypeScript — single binary, zero dependencies
- **Workload runtime**: flox.dev (reproducible Nix environments) or OCI containers (Docker/Podman)
- **Proxy**: Caddy — reverse proxy, load balancing, auto SSL
- **State**: Embedded SQLite — deployments, releases, health logs, locks
- **Secrets**: AES-256-GCM — encrypted at rest, injected at deploy
- **Agent**: Optional (TypeScript compiled with `bun build --compile`), agentless by default

## Key Features

- **Multi-server deployment**: Parallel SSH with configurable concurrency
- **3 strategies**: Rolling, Blue-Green, Canary
- **3 environments**: Staging, Test, Production (configurable in `dnx.yaml`)
- **Reproducible builds**: flox.dev (lockfile) or Docker/OCI (BuildKit, no daemon)
- **Built-in Caddy**: Reverse proxy, load balancing, auto SSL
- **Managed secrets**: `dnx secrets set DATABASE_URL=...` → AES-256-GCM → auto-injected
- **Health checks**: HTTP/TCP/command probes, state machine, SQLite history
- **Rollback**: `dnx rollback --workload web` → previous version
- **Lock system**: SQLite + remote locks to prevent concurrent deployments
- **Optional agent**: Process supervisor, log buffer, restart policy
