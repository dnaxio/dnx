# Quickstart

## 1. Initialize a project

```bash
mkdir my-app && cd my-app
dnx init --name my-app
```

This creates a `dnx.yaml` file with a base configuration.

## 2. Set up your variables

Create a `.env` file at the project root:

```bash
# .env
PROD_HOST=my-server.example.com
STAGING_HOST=staging.example.com
```

DNX automatically loads these variables. Use `$PROD_HOST` in your `dnx.yaml`.

## 3. Configure your servers

Edit `dnx.yaml` to add your servers:

```yaml
environments:
  staging:
    servers:
      - name: staging-1
        host: 10.0.1.10
        user: deploy
        port: 22
        key_path: ~/.ssh/id_ed25519
```

## 4. Define your workload

```yaml
workloads:
  - name: web
    repo_url: git@github.com:org/web.git
    branch: main
    runtime:
      type: flox
      flox_env: web-env
      start_cmd: "node dist/server.js"
      ports:
        - 3000
      env:
        NODE_ENV: "production"
        DATABASE_URL: "${secret:DATABASE_URL}"
    build:
      steps:
        - run: npm ci
        - run: npm run build
    health:
      type: http
      endpoint: /health
```

## 5. Set up secrets

```bash
dnx secrets init
dnx secrets set DATABASE_URL=postgres://user:pass@host:5432/db --env production
```

## 6. Deploy

```bash
# Build (runs build steps via flox)
dnx build web

# Deploy a single workload to staging
dnx deploy web --env staging --tag v1

# Deploy all workloads
dnx deploy --all --env staging --tag v1

# Deploy to production (blue-green)
dnx deploy web --env production --tag v1 --strategy blue-green
```

## 7. Verify

```bash
# Workload status
dnx workload status web --env staging

# Logs
dnx workload logs web --env staging --follow

# Health check
dnx health check web --env staging
```
